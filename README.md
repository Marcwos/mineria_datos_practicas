## üî∞ PARTE 1: Series de Pandas (practica1.ipynb & temperaturas.ipynb)

### ¬øQu√© es una Serie?
Una Serie es una estructura de datos unidimensional (como una columna de Excel) con √≠ndices personalizados.

### Creaci√≥n de Series
```python
# Serie b√°sica
ventas = pd.Series([5, 10, 15, 30], 
                   name="Ventas", 
                   index=["Camisa", "Pantalon", "Zapatos", "Calcetas"])

temperaturas = pd.Series([28, 30, 27, 29, 31], 
                         name="Temperaturas", 
                         index=["Lunes", "Martes", "Miercoles", "Jueves", "Viernes"])
```

### Operaciones con Series
```python
# Operaciones matem√°ticas entre Series
total = ventas * cantidad  # Multiplica elemento por elemento
total.name = "Total"       # Asignar nombre a la Serie

# Verificar valores
9 in cantidad.values       # Verifica si 9 est√° en los valores
```

### Acceso a elementos de Series
```python
# Por √≠ndice personalizado
temperaturas["Miercoles"]                    # Un elemento
temperaturas["Lunes":"Miercoles"]            # Rango (inclusivo)
cantidad["Camisa":"Zapatos"]                 # Rango de productos

# Por posici√≥n num√©rica
cantidad[0:3]                                # Primeros 3 elementos (0, 1, 2)

# Por condici√≥n
temperaturas[temperaturas > 28]              # Temperaturas mayores a 28
cantidad[cantidad >= 5]                      # Cantidades >= 5
cantidad > 5                                 # Retorna booleanos
```

### Propiedades de Series
```python
cantidad.name          # Nombre de la Serie
cantidad.index         # √çndices de la Serie
cantidad.values        # Valores de la Serie
```

### Modificaci√≥n y eliminaci√≥n de Series
```python
# Actualizar valores
temperaturas["Miercoles"] = 25

# Eliminar elementos
del temperaturas["Viernes"]
```

### Convertir Serie a DataFrame
```python
df = pd.DataFrame({"temperaturas": temperaturas})
```

## üìù Conceptos Clave y Diferencias Importantes

### üîç Serie vs DataFrame
| Serie | DataFrame |
|-------|-----------|
| Unidimensional (1 columna) | Bidimensional (varias columnas) |
| Tiene √≠ndice y valores | Tiene √≠ndice, columnas y valores |
| Similar a una lista con √≠ndices | Similar a una tabla de Excel |

### üéØ loc vs iloc
| **loc** | **iloc** |
|---------|----------|
| Selecci√≥n por **etiquetas** | Selecci√≥n por **posici√≥n num√©rica** |
| `df.loc["Camisa"]` | `df.iloc[0]` |
| `df.loc[:, "precio"]` | `df.iloc[:, 1]` |
| Inclusivo en rangos: `[0:3]` incluye 0,1,2,3 | Exclusivo en rangos: `[0:3]` incluye 0,1,2 |

### üî¢ M√©todos importantes para encontrar valores

```python
# Encontrar √≠ndice del m√°ximo/m√≠nimo
df["Salario"].idxmax()        # √çndice del salario m√°ximo
df["Salario"].idxmin()        # √çndice del salario m√≠nimo

# Obtener el valor m√°ximo/m√≠nimo
df["Salario"].max()           # Valor m√°ximo
df["Salario"].min()           # Valor m√≠nimo

# Obtener toda la fila del m√°ximo
df.loc[df["Salario"].idxmax()]
```

### ‚öôÔ∏è Operadores para filtros
- **`&`**: AND (Y) - Ambas condiciones deben cumplirse
- **`|`**: OR (O) - Al menos una condici√≥n debe cumplirse
- **`~`**: NOT (Negaci√≥n) - Invierte la condici√≥n

**‚ö†Ô∏è IMPORTANTE**: Usa `&` y `|`, NO `and` y `or` en pandas. Siempre usa par√©ntesis en cada condici√≥n.

```python
# ‚úÖ CORRECTO
df[(df["Edad"] > 23) & (df["Salario"] > 1200)]

# ‚ùå INCORRECTO
df[df["Edad"] > 23 and df["Salario"] > 1200]
```

### üìä Funciones de agregaci√≥n comunes
- `mean()`: Media/promedio
- `sum()`: Suma total
- `count()`: Conteo de valores
- `max()`: Valor m√°ximo
- `min()`: Valor m√≠nimo
- `std()`: Desviaci√≥n est√°ndar
- `var()`: Varianza
- `median()`: Mediana
- `nunique()`: N√∫mero de valores √∫nicos
- `unique()`: Lista de valores √∫nicos

### üí° Errores comunes a evitar

1. **Usar `and`/`or` en lugar de `&`/`|`** en filtros
   ```python
   # ‚ùå MAL
   df[df['edad'] > 20 and df['edad'] < 30]
   
   # ‚úÖ BIEN
   df[(df['edad'] > 20) & (df['edad'] < 30)]
   ```

2. **Olvidar par√©ntesis en condiciones m√∫ltiples**
   ```python
   # ‚ùå MAL
   df[df['a'] > 5 & df['b'] < 10]
   
   # ‚úÖ BIEN
   df[(df['a'] > 5) & (df['b'] < 10)]
   ```

3. **Confundir loc e iloc**
   ```python
   df.loc[0]        # Busca √≠ndice etiquetado como 0
   df.iloc[0]       # Busca primera fila por posici√≥n
   ```

4. **No asignar el resultado de modificaciones**
   ```python
   # ‚ùå MAL (no guarda cambios)
   df.dropna()
   
   # ‚úÖ BIEN
   df = df.dropna()
   # o usar inplace=True
   df.dropna(inplace=True)
   ```

## üéØ Tips y Trucos para el Examen

### üéì Puntos clave para recordar

1. **Series son unidimensionales**, DataFrames son bidimensionales
2. **loc usa etiquetas**, iloc usa posiciones num√©ricas
3. **Usar `&` y `|`** para condiciones, NO `and` y `or`
4. **Siempre usar par√©ntesis** en cada condici√≥n de filtros m√∫ltiples
5. **`apply()` con lambda`** es muy √∫til para transformaciones complejas
6. **`groupby()` + `agg()`** para an√°lisis agrupados con m√∫ltiples estad√≠sticas
7. **Siempre verificar valores nulos** antes de hacer an√°lisis: `df.isnull().sum()`
8. **`idxmax()` y `idxmin()`** devuelven el √≠ndice, no el valor
9. **Los rangos con loc son inclusivos**, con iloc son exclusivos
10. **`inplace=True`** modifica el DataFrame original, sin asignar
